⚙️ Global Config v1.2.0

ตั้งค่า DataGrid ครั้งเดียวจาก app.config.ts — ทุก <app-datagrid> ในทุกหน้า ได้ config นี้อัตโนมัติ เทียบเท่า dxDataGrid.defaultOptions() ของ DevExtreme

✅ Instance options ชนะเสมอ — ถ้าหน้าไหนต้องการ config ต่างออกไป ระบุใน [options] ได้ปกติ โดยไม่กระทบหน้าอื่น 
// app.config.ts — ตั้งครั้งเดียว ใช้ทุกหน้า
import { provideDatagridDefaults } from '@darkoum/ng-datagrid';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient(),
    provideAnimationsAsync(),

    // ─── Global DataGrid defaults ─────────────────
    provideDatagridDefaults({
      showBorders:           true,
      rowAlternationEnabled: true,
      hoverStateEnabled:     true,
      allowColumnResizing:   true,
      columnMinWidth:        50,
      paging:    { enabled: true, pageSize: 15 },
      pager:     { showPageSizeSelector: true,
                   allowedPageSizes: [15, 20, 30, 0],
                   showInfo: true },
      selection: { mode: 'single' },
      editing:   { popup: { showTitle: true } },
    }),
  ],
};
// any-page.component.ts — ระบุแค่สิ่งที่ต่างออกไป
gridOptions: DataGridOptions = {
  keyExpr:    'studentid',
  dataSource: { store: this.students },
  columns: [ ... ],
  // ไม่ต้องระบุ showBorders, paging, pager ซ้ำ
  // ถ้าหน้านี้ต้องการ pageSize ต่าง — override ได้
  paging: { pageSize: 20 },   // override default 15 → 20 เฉพาะหน้านี้
};
Default OptionTypeคำอธิบาย
showBordersbooleanแสดงเส้นกรอบตาราง
rowAlternationEnabledbooleanสีสลับแถว
hoverStateEnabledbooleanhighlight เมื่อ hover
allowColumnResizingbooleanปรับความกว้าง column ได้
columnMinWidthnumberความกว้างขั้นต่ำ (px)
pagingPagingConfig{ enabled, pageSize }
pagerPagerConfig{ showPageSizeSelector, allowedPageSizes, showInfo }
selectionSelectionConfig{ mode: 'none'|'single'|'multiple' }
editingEditingConfigดู Editing section สำหรับ options ทั้งหมด
💡 Merge logic: nested objects เช่น paging, pager, selection, editing จะถูก deep-merge — instance property ชนะ default ทีละ key ไม่ใช่ replace ทั้ง object

🅐 วิธีผ่าน GitHub Packages

ทำครั้งเดียว ทีมโหลดเองได้ตลอด

ฝั่งคุณ (ผู้ publish)

1. สร้าง GitHub Repository แล้ว push โค้ดขึ้น

# ใน demo/
git init
git remote add origin https://github.com/vn-org/ng-datagrid.git
git add .
git commit -m "feat: initial library release"
git push -u origin main

2. สร้าง Personal Access Token (PAT)

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
เลือก scope: ✅ write:packages   ✅ read:packages
Copy token ที่ได้ไว้

3. Login GitHub Packages

npm login --registry=https://npm.pkg.github.com
# Username: your-github-username
# Password: ghp_xxxxxxxxxxxx  ← PAT ที่เพิ่งสร้าง
# Email: your@email.com

4. Build & Publish

npm run build:lib
cd dist/ng-datagrid
npm publish
# ✅ @darkoum/ng-datagrid@1.0.0 ขึ้น GitHub Packages แล้ว

ฝั่งคนที่จะใช้ (ทำในโปรเจคตัวเอง)

1. สร้าง PAT ของตัวเอง (scope: read:packages เท่านั้น)

2. เพิ่ม .npmrc ในโปรเจค

@darkoum:registry=https://npm.pkg.github.com

3. ติดตั้ง

Validation v1.2.0

ตรวจสอบความถูกต้องของข้อมูลผ่าน validate() method — รองรับ 5 rule types, แสดงขอบแดง + ข้อความ error อัตโนมัติ

ValidationRule Types

typeตัวเลือกคำอธิบาย
requiredmessage?ห้ามว่าง — รองรับ string, array, null
stringLengthmin?, max?, message?จำกัดความยาวตัวอักษร
rangemin?, max?, message?จำกัดค่าตัวเลข
patternpattern (RegExp|string), message?ตรวจ format ด้วย Regular Expression
customvalidationCallback: (v)=>boolean, message?เขียน logic ตรวจเองได้

FormComponent API

Methodคืนค่าคำอธิบาย
validate()ValidationResultตรวจทุก field — แสดง error border + message ทันที
clearErrors()voidล้าง error ทั้งหมดออก
 
// 1. กำหนด rules ใน FormItem
formItems: FormItem[] = [
  {
    dataField: 'studentcode',
    label: { text: 'รหัสนักศึกษา' },
    isRequired: true,           // แสดง * + required check อัตโนมัติ
    validationRules: [
      { type: 'stringLength', min: 8, max: 10,
        message: 'ต้องมี 8-10 ตัวอักษร' },
      { type: 'pattern', pattern: /^[0-9]+$/,
        message: 'ต้องเป็นตัวเลขเท่านั้น' },
    ],
  },
  {
    dataField: 'gpa',
    validationRules: [
      { type: 'range', min: 0, max: 4 },
    ],
  },
  {
    dataField: 'email',
    validationRules: [
      { type: 'pattern',
        pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
        message: 'รูปแบบอีเมลไม่ถูกต้อง' },
    ],
  },
  {
    dataField: 'score',
    validationRules: [
      { type: 'custom',
        validationCallback: v => v % 0.5 === 0,
        message: 'คะแนนต้องเป็นทวีคูณของ 0.5' },
    ],
  },
];

// 2. เรียก validate() ผ่าน ViewChild
@ViewChild('myForm') formRef!: FormComponent;

save() {
  const result = this.formRef.validate();
  // result.isValid        — true / false
  // result.brokenRules    — [{ dataField, message }, ...]

  if (result.isValid) {
    // บันทึกข้อมูล
  }
}

// 3. ล้าง error ทั้งหมด
this.formRef.clearErrors();
💡 พฤติกรรม: error border + ข้อความแดงปรากฏใต้ field ทันทีที่ validate() ถูกเรียก — เมื่อผู้ใช้แก้ไข field นั้น error จะหายอัตโนมัติ — ลอง 🛡 Validation Demo

📆 ThaiDate Pipe

แปลงวันที่เป็นปีพุทธศักราช (พ.ศ.)

Formatผลลัพธ์
 
{{ myDate | thaiDate }}              <!-- 25 มกราคม 2568       -->
{{ myDate | thaiDate:'short' }}      <!-- 25 ม.ค. 68           -->
{{ myDate | thaiDate:'long' }}       <!-- วันศุกร์ที่ 25 มกราคม พ.ศ. 2568 -->
{{ myDate | thaiDate:'shortDate' }} <!-- 25/01/2568           -->
Formatตัวอย่างผลลัพธ์
short25 ม.ค. 68
mediumDate25 ม.ค. 2568
medium25 มกราคม 2568
longวันเสาร์ที่ 25 มกราคม พ.ศ. 2568
shortDate25/01/2568
shortDateTime25 ม.ค. 68 10:30
timeOnly10:30
